# Semantic Type to Strategy Mapping

This section details the Semantic Type to Strategy Mapping.

## Feature

| Section Type | Strategy | Description |
|---|---|---|
| purpose | completeness | Is the feature's purpose clearly stated? |
| functional_requirements | completeness | Are all functional requirements enumerated? |
| business_rules | completeness | Are business rules unambiguous and testable? |
| constraints | completeness | Are constraints specific and justified? |
| success_criteria | completeness | Are success criteria measurable? |
| inputs | completeness | Are all inputs specified with types and ranges? |
| outputs | completeness | Are all outputs specified with formats? |
| dependencies | completeness | Are dependencies identified with versions? |
| non_goals | completeness | Are explicit boundaries stated? |
| future_extensions | completeness | Are future scope items documented? |
| traceability | consistency | Do traceability links resolve to valid targets? |

## Architecture

| Section Type | Strategy | Description |
|---|---|---|
| purpose | completeness | Is the architectural purpose clear? |
| system_overview | completeness | Is the system context described? |
| component_model | completeness | Are all components identified with responsibilities? |
| communication_paths | completeness | Are all communication paths documented? |
| data_flow | completeness | Is data flow through the system described? |
| security_considerations | completeness | Are security concerns addressed? |
| rationale | completeness | Are architectural decisions justified? |
| constraints | completeness | Are architectural constraints explicit? |
| traceability | consistency | Do architectural decisions trace to requirements? |
