# samgraha registry

## Purpose

Manage this repository's local registry of known repositories (stored in `.samgraha/registry.db`) — a different registry from the compiled-knowledge `knowledge.db` — used for cross-repo dependency resolution.

## Content

### Synopsis

```bash
samgraha registry <register|unregister <uuid>|sync|refresh|status|list|resolve <mode>>
```

### Subcommands

| Subcommand | Description |
|------------|-------------|
| `register` | Register the current repository, reading its `.samgraha/manifest.json` (requires `compile` to have run first) |
| `unregister <uuid>` | Unregister a repository by its UUID |
| `sync` | Synchronize cached dependency metadata (`.meta` files) from registered repositories' manifests |
| `refresh` | Refresh all cached dependency metadata |
| `status` | Show registry status |
| `list` | List all registered repositories |
| `resolve <mode>` | Resolve dependencies for runtime (mode e.g. `runtime`) |

### Description

The repository registry tracks repositories known on this machine, distinct from the per-repository `knowledge.db`. Each registered repository is identified by the manifest generated by `samgraha compile` (`.samgraha/manifest.json`), which declares its identity, exports, and dependencies.

### Registry Storage

Registration data is stored in `.samgraha/registry.db` (SQLite), per repository — not in workspace configuration.

### Examples

```bash
# Register the current repository (must be compiled first)
samgraha compile
samgraha registry register

# List registered repositories
samgraha registry list

# Unregister a repository
samgraha registry unregister 4ee433b6-62d0-4683-9a9e-2c58549b6332

# Sync dependency metadata
samgraha registry sync

# Resolve dependencies for runtime
samgraha registry resolve runtime
```
