# Workflows

## Purpose

Workflows are phased project plans generated by knowledge systems. They define the sequence of operations (generate, audit, fix, verify) that transform a repository from its current state to a desired state.

## What Workflows Are

A workflow is a plan with ordered phases, each targeting specific domains and capabilities. Workflows are generated by the system's `plan_generation` script and stored in the `system_plans` table.

## Workflow Structure

```
Workflow
  ├── Phase 1: Generate (compile documentation)
  ├── Phase 2: Audit (validate all domains)
  ├── Phase 3: Fix (address findings)
  ├── Phase 4: Verify (re-audit, check gates)
  └── Phase 5: Report (generate summary)
```

## Phase Types

| Phase Type | What It Does | System Script |
|---|---|---|
| `generate` | Compile documentation | `init` or built-in |
| `audit` | Run validation checks | `validate` |
| `fix` | Apply fixes to findings | `validate` (re-audit) |
| `verify` | Re-run checks, compare thresholds | `validate` |
| `report` | Generate summary reports | `report` |

## Phase Dependencies

Phases can declare dependencies on other phases:

```yaml
phases:
  - id: audit-doc
    name: "Audit Documentation"
    depends_on: []  # runs first

  - id: fix-doc
    name: "Fix Documentation"
    depends_on: ["audit-doc"]  # runs after audit

  - id: verify
    name: "Verify Fixes"
    depends_on: ["fix-doc"]  # runs after fix
```

## Prerequisites

Before a phase executes, samgraha checks its prerequisites:

- All `depends_on` phases must be completed
- TTL expiry rules must not be violated
- Head commit must match (if configured)

If prerequisites are not met, the phase is blocked and returns a blocked-precondition response.

## Init Plans

Systems declare their workflows via `init` plans in `system.yaml`:

```yaml
init:
  use_cases:
    - id: new-project
      phases:
        - name: "Generate"
          step: "Create initial documentation from templates"
        - name: "Validate"
          step: "Run all validation checks"
        - name: "Fix"
          step: "Address any findings"
    - id: docs-audit
      phases:
        - name: "Audit"
          step: "Run documentation validation"
        - name: "Fix"
          step: "Address documentation findings"
```
