# Design Domain Guide

## Purpose

How to write Design documentation — reusable design language, principles, and UX standards.

## Content

### Purpose of Design Docs

Design documentation defines the reusable design system: principles, UX guidelines, and accessibility standards that apply across all features.

### Required Sections

| Section | Required | Description |
|---------|----------|-------------|
| Design Principles | Yes | Core design philosophy |
| UX Principles | Yes | User experience guidelines |
| Accessibility | Yes | Accessibility standards |

### Design vs Feature Design

- **Design** — Reusable standards that apply to all features
- **Feature Design** — Specific design decisions for one feature

### Writing Tips

- Design principles should be timeless, not tied to current trends.
- UX guidelines should be actionable (developers should know exactly what to do).
- Accessibility standards should reference WCAG or similar recognized standards.
- Avoid feature-specific workflows (they belong in Feature Design).
