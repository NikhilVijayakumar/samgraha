# Philosophy Domain Guide

## Purpose

How to write Philosophy documentation — product philosophy and design principles.

## Content

### Purpose of Philosophy Docs

Philosophy documents capture the core principles, values, and trade-offs that guide all product and engineering decisions.

### Required Sections

| Section | Required | Description |
|---------|----------|-------------|
| Principles | Yes | Core design and engineering principles |
| Values | Yes | What the product values |
| Trade-offs | Yes | Deliberate trade-offs made |

### Philosophy vs Vision

- **Vision** — What we build and for whom
- **Philosophy** — How we think about building it

### Writing Tips

- Principles should be opinionated and specific enough to guide decisions.
- Trade-offs should be honest — acknowledge what was sacrificed.
- Philosophy changes rarely. When it does, it represents a significant shift.
- Avoid technology-specific content (belongs in Engineering).
